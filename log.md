# LOG

目前这个项目我是想开发一个基于Github Gists的针对vless、vmess等这些网络代理协议订阅或者单个节点的纯前端实现的管理页面。主要想实现的关键功能有如下几点：
1. 使用Github Gists 作为存储(类似数据库功能)
2. 可以对单条或者多条节点信息或者订阅信息进行聚合，形成聚合后的一条订阅信息
3. 可以在聚合结果之前，可以针对单个或者多个节点进行编辑，比如重命名，剔除特定节点等等
4. 整体功能类似于Sub-Store，但是本项目不需要后端API服务，计算纯依赖于当前用户浏览器，且数据存储在Github Gists
我对技术栈的要求如下：
1. 使用最新版的svelte和tailwindcss构建
2. 引入biome这样的标准的lint工具
3. 包依赖工具使用pnpm

请先帮我搭建一下基础框架吧，请注意，如果执行shell的时候需要访问外部网络，则需要先执行 setproxy 命令，设置 http://127.0.0.1:20172 的代理，或者直接执行命令的时候直接export这个代理
